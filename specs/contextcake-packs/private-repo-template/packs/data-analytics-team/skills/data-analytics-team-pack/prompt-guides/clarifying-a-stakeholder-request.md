# Clarifying a Stakeholder Request {#clarifying-a-stakeholder-request}

Prompt-level guidance for how an agent should actually run the Clarify
phase in conversation — not the workflow's structure, but the wording and
judgment calls that make it feel natural instead of interrogative.

## What it covers {#what-it-covers}

How to ask for the minimum clarification needed rather than front-loading
every possible question, how to propose a reasonable default instead of
asking when the team's conventions already answer it, and how to phrase
questions so a non-technical stakeholder can answer them without
translation.

## Why prompt guidance separate from the workflow {#why-it-matters}

The workflow says what to capture; this guide says how to get it out of a
busy stakeholder in one or two exchanges instead of a lengthy back-and-forth
that makes them regret asking.
