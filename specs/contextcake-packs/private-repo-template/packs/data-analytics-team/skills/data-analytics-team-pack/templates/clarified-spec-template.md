# Clarified Spec Template {#clarified-spec-template}

The output artifact of the Clarify phase — a fill-in-the-blanks structure
that turns a stakeholder's request into something the Build phase can
work from without guessing.

## What it captures {#what-it-captures}

Fields for the metric's precise definition, time window, segment, grain,
and the requester's actual decision they're trying to make — the thing
that determines whether the analysis is even answering the right
question.

## Why a template instead of free text {#why-a-template}

Free-text clarification drifts; a template forces the same fields every
time, which is what makes the Validate phase possible later — you can't
reconcile a result against a source of truth if you don't know precisely
what was asked.
