# Changelog

## 0.1.0 — 2026-07-06

Initial content release. The pack moves from OKF scaffold to full authored content:

- **Overview** — pack purpose, audience, and the one problem it solves
  (`overview/pack-purpose.md`).
- **Glossary** — six shared terms: source of truth, stakeholder request, clarified spec,
  metric definitions, validation pass, insight summary (`glossary/`).
- **Workflows** — the four-phase hero workflow (clarify, build, validate, deliver) plus
  its three phase-level breakouts: request clarification, build-and-validate, and
  insight delivery (`workflows/`).
- **Templates** — the clarified-spec, validation-checklist, and insight-summary
  templates used on every request (`templates/`).
- **Examples** — three worked examples, including the canonical Northwind MRR-trend
  request, grounding the templates in a full end-to-end request (`examples/`).
- **Prompt guides** — three guides for running pack workflows with an AI assistant in
  the loop, starting with clarifying a stakeholder request (`prompt-guides/`).
- **Policies** — three policies covering source-of-truth precedence and related
  change-control rules for when systems disagree or definitions shift
  (`policies-and-rules/`).
- **Tool guides** — four guides for using this pack with Claude Code, ChatGPT, Cursor,
  and GitHub Copilot (`tool-guides/`).

All examples use the fictional company Northwind; all figures are synthetic and
illustrative.
