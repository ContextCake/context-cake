# Build and Validate {#build-and-validate}

Covers the middle two phases of the hero workflow: producing the analysis
against a clarified spec, then checking it before it goes anywhere near a
stakeholder.

## Build {#build}

Guidance on working from the clarified spec rather than re-interpreting
the original request, including how to flag when the data available
doesn't actually support what was asked for.

## Validate {#validate}

A reconciliation pass against the team's source of truth — does the
number match what the system of record says, within a reasonable
tolerance, and if not, why. This is where silent errors get caught before
delivery instead of after.

_The complete version ships with the pack._
