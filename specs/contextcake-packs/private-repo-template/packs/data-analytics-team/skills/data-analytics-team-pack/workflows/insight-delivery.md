# Insight Delivery {#insight-delivery}

Covers the final phase of the hero workflow: turning a validated result
into something a stakeholder can actually act on.

## What good delivery looks like {#what-good-delivery-looks-like}

Not just a number — a stated confidence level, the caveats that matter
(small sample, partial period, known data gaps), and a recommended next
action. This workflow defines the shape of that package and when each
piece is required versus optional.

## Why it matters {#why-it-matters}

A number without context gets over-trusted or ignored. This is the phase
that decides whether the stakeholder actually uses the insight — the
payoff for the three phases that came before it.
